## Versioning

`MAJOR.MINOR.PATCH`

- **MAJOR** — breaking changes to the public API or significant architectural rewrites  
- **MINOR** — new tools, new features, non-breaking changes  
- **PATCH** — bug fixes, performance improvements, copy changes

## Contributing

Each PR should include a changelog entry in `changelogs/vX.Y.Z.md` under the appropriate version heading before merging.
